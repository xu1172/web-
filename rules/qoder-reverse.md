# Qoder 逆向核心约束

> 本文件只定义不可突破的硬约束。具体操作细节见 [CLAUDE.md](../../../CLAUDE.md)。

## 1. 核心原则

- **协议优先**：目标是用 `requests`/`axios` 直接请求，MCP 仅用于调试定位。禁止 Selenium/Playwright/Puppeteer 做数据采集
- **真实数据优先**：一切以真实抓包为准。禁止 `...`/`xxx`/`example` 占位，禁止伪造 sign/cookie/token 验收。无真实样本时必须明说"未实证"
- **禁止猜测**：加密方式必须动态调试实证后再编码，严禁凭经验猜
- **Python 仅用项目 venv**

## 2. Phase 准入门槛

| Phase | 进入下一 Phase 前必须完成 |
|-------|--------------------------|
| 0 情报收集 | 默认执行；用户明确说"跳过"可省略。结果落 `docs/notes.md` |
| 1 流量分析 | 确认接口 URL/Method/Content-Type + 至少 1 组真实请求/响应 + 标记哪些字段需逆向 |
| 2 加密定位 | 加密函数源码位置实证 + 浏览器输入/输出对照 + 本地复现对照 |
| 3 Plan | 产出 `plan.md`（接口表/参数表/加密实证/实现方案）+ **用户确认**。**未经确认严禁写代码** |
| 4 实现 | 最小可复现代码 + 真实样本端到端验证。未完成端到端只能标记为"阶段性结果" |

## 3. 调试路线

```text
需要指纹保护（有风控/验证码/Cloudflare）？
  ├─ 是 → 路线A (AdsPower) 或 路线C (CDP桥)
  └─ 否 → 路线B (js-reverse-mcp)
```

`adspower-browser` 和 `js-reverse-mcp` 是独立浏览器进程，不能共享登录态/Hook/断点。

- **路线 A**：指纹安全，但无断点/源码搜索 → 仅适合轻量 Hook
- **路线 B**：完整调试（断点/搜索/调用栈），但普通 Chrome 可能被风控
- **路线 C**：AdsPower 指纹 + Python CDP 可编程调试（金标准）

## 4. 禁止清单

| 禁止 | 替代 |
|------|------|
| 浏览器自动化采集 | `requests`/`axios` 直接请求 |
| 猜测性编码 | 动态调试确认后编码 |
| 跳过 Plan 阶段 | Phase 3 Plan + 用户确认 |
| 占位符/伪造值验收 | 真实样本端到端验证 |
| 重复无效操作 | 拿到线索追到底或换方法 |

## 5. 必须产出

每个任务：Python 协议脚本 + 加密/签名模块 + 参数文档 + 真实样本验证证据


## 7. 验证码厂商速查

| 厂商 | 常见参数 |
|------|---------|
| 极验 | `gt`, `challenge`, `w` |
| 网易易盾 | `token`, `acToken`, `data` |
| 腾讯防水墙 | `aid`, `cap_cd`, `sess` |
| 阿里验证 | `nc_token`, `bizId`, `sig` |
| 数美 | `riskToken`, `request_id` |
| 瑞数 | `_sd`, `_fd` 类动态字段 |
